# Workflow Rules

## Rule 1: No Pretending

If something is confusing, say so.

## Rule 2: If It Matters, Put It In Git

Important project knowledge should live in the repository, not only in chat history.

## Rule 3: Public Stays Generic

Public repositories and commercial builds must not include private proposal content.

## Rule 4: Small Commits

Each commit should represent one focused change.

## Rule 5: Main Should Stay Usable

Avoid committing broken work to `main` once the project becomes playable.

## Rule 6: Parking Lot, Not Scope Creep

Good ideas that do not fit the current milestone go into the Parking Lot.
