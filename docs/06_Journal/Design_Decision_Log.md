# Design Decision Log

## DDL-001: Use Godot

Decision:
Use Godot 4 Standard Edition.

Reason:
Godot is lightweight, open-source, strong for 2D, and beginner-friendly.

## DDL-002: Use 32x32 Pixel Art

Decision:
Use cozy 32x32 character sprites.

Reason:
This balances expressiveness, animation workload, and beginner feasibility.

## DDL-003: Split Into Three Projects

Decision:
Build StorySprout, First Bloom, and Playable Memory separately.

Reason:
This keeps public work portfolio-safe, creates a commercial path, and protects the private proposal content.

## DDL-004: Public Projects Exclude Proposal Content

Decision:
No proposal, real names, or personal relationship details in public repositories/releases.

Reason:
Protects privacy and keeps public projects commercially reusable.

## DDL-005: Use Git From Day One

Decision:
Version control all important files from the start.

Reason:
Creates professional habits, protects the project, and builds a visible development history.

## DDL-006: Use Markdown for Living Docs

Decision:
Keep living project documents in Markdown.

Reason:
Markdown works well with GitHub, is easy to diff, and is standard in software projects.
